## 📋 重要说明

### 字段命名规范（统一标准）

1. **数据库字段**: 统一使用 `snake_case`（下划线命名）
2. **后端API请求字段**: 前端发送时，**统一使用 `snake_case`**（后端同时支持 `camelCase` 和 `snake_case` 以兼容旧代码）
3. **后端API响应字段**: **统一使用 `snake_case`**（与数据库一致）

### 关键原则

- ✅ **前端发送请求时，统一使用 `snake_case`**
- ✅ **前端接收响应时，统一使用 `snake_case`**
- ✅ **后端会同时支持 `camelCase` 和 `snake_case`**（兼容处理，但建议前端使用 `snake_case`）
- ⚠️ **外键字段（如 `profile_id`）必须存在于数据库中，否则会触发外键约束错误**

## 🚨 常见问题和解决方案

### 1. 外键约束错误

**问题**: `profile_id` 不存在于 `profiles` 表中

**错误特征**:
- HTTP状态码: `500`
- 错误信息包含: `foreign key constraint` 或 `profile_id_fkey` 或 `analysis_sessions_profile_id_fkey`

**解决方案**:
```typescript
// 前端处理逻辑
async function saveAnalysisSessionWithFallback(
  profileId: string,
  sessionData: any,
  userId: string
) {
  try {
    // 第一次尝试：使用原始profileId
    return await saveAnalysisSession({ profileId, sessionData });
  } catch (error: any) {
    // 检测外键约束错误
    const isForeignKeyError = 
      error.status === 500 && 
      error.message && (
        error.message.includes('foreign key constraint') || 
        error.message.includes('profile_id_fkey') ||
        error.message.includes('analysis_sessions_profile_id_fkey')
      );
    
    if (isForeignKeyError && profileId !== userId) {
      // 外键约束错误，使用userId作为profileId重试
      console.warn(`[分析会话] profileId无效，使用userId作为profileId:`, {
        originalProfileId: profileId,
        fallbackProfileId: userId
      });
      
      return await saveAnalysisSession({
        profileId: userId,  // 使用userId作为profileId（对应"我的命盘"）
        sessionData: {
          ...sessionData,
          _metadata: {
            ...(sessionData._metadata || {}),
            originalProfileId: profileId,  // 记录原始profileId
            fallbackReason: 'profile_id_not_found'
          }
        }
      });
    }
    
    // 其他错误，直接抛出
    throw error;
  }
}
```

**后端改进**（✅ 已实现）:
- ✅ 在保存前验证 `profileId` 是否存在
- ✅ 如果不存在，返回明确的错误信息（400 Bad Request），而不是外键约束错误（500）
- ✅ 如果 `profileId` 等于 `userId`，则认为是"我的命盘"，直接通过验证

**后端验证逻辑**:
```typescript
// 如果 profileId 等于 userId，则认为是"我的命盘"，直接通过
if (profileId !== userId) {
  const profileCheck = await pool.query(
    'SELECT id FROM public.profiles WHERE id = $1',
    [profileId]
  );
  
  if (profileCheck.rows.length === 0) {
    throw new Error(`参数错误：命盘ID不存在（profileId: ${profileId}）。提示：如果这是"我的命盘"，请使用 userId 作为 profileId`);
  }
}
```

**错误响应**:
```json

}
```
