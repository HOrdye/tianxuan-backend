# 合盘功能 - 前端适配与校验文档

本文档供前端对接合盘（含智能定盘、深度合盘分析）接口时做适配与联调校验使用。所有接口需认证：`Authorization: Bearer <token>`。

---

## 1. 接口总览

| 能力 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 创建合盘 | POST | `/api/astrology/compatibility` | 支持定盘元数据 rectification |
| 创建分析任务 | POST | `/api/astrology/compatibility/analyze/task` | 返回 taskId，供 GET stream 使用 |
| 分析流（SSE） | GET | `/api/astrology/compatibility/analyze/stream?taskId=xxx` | 命名事件：start/chunk/json_block/done/error |
| 深度合盘分析（流式） | POST | `/api/astrology/compatibility/analyze` | 传统 SSE：data.content + [DONE] |
| 合盘列表 | GET | `/api/astrology/compatibility` | limit/offset/keyword |
| 合盘详情 | GET | `/api/astrology/compatibility/:id` | 含定盘字段 |
| 删除合盘 | DELETE | `/api/astrology/compatibility/:id` | - |

响应约定：成功 `{ success: true, data: ... }`，失败 `{ success: false, error: string, message?: string }`。除流式接口外，`data` 字段名均为 **snake_case**。

---

## 2. 创建合盘存档

**请求**: `POST /api/astrology/compatibility`

**Body**（JSON，字段支持 camelCase 或 snake_case）:

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| chartA / chart_a | object | 是 | 完整 ZiweiChart |
| chartB / chart_b | object | 是 | 完整 ZiweiChart |
| name | string | 是 | 合盘名称 |
| notes | string | 否 | 备注 |
| tags | string[] | 否 | 标签 |
| rectification | object | 否 | 定盘元数据，见下表 |

**rectification**（可选）:

| 字段 | 类型 | 说明 |
|------|------|------|
| method | string | 仅接受 `triangulation` \| `mbti` \| `manual` |
| inferredHour / inferred_hour | string | 推导时辰，如 `chou` |
| confidence | number | 0–100，后端会夹到该范围 |
| inferenceData / inference_data | object | 任意 JSON（tags、matchedStars 等） |

**成功响应** (200):
```json
{
  "success": true,
  "data": {
    "id": "uuid",
    "user_id": "uuid",
    "name": "合盘名称",
    "created_at": "2025-02-12T00:00:00.000Z"
  }
}
```

**校验点**: 传 chartA/chartB/name 可创建；带 rectification 时 method 为三者之一，confidence 0–100；响应用 data.id 做跳转或列表刷新。

## 3. 深度合盘分析（推荐：POST 任务 + GET 流）

**步骤 1 - 创建任务**: `POST /api/astrology/compatibility/analyze/task`  
**Body**: `{ chartA, chartB, relationshipType? }`（支持 chart_a/chart_b、relationship_type）  
**响应** (200): `{ "success": true, "data": { "taskId": "uuid" } }`

**步骤 2 - 监听流**: `GET /api/astrology/compatibility/analyze/stream?taskId=<uuid>`  
- EventSource 无法带 Header，认证通过 Query 传参：`?taskId=xxx&token=<access_token>` 或 `access_token=<access_token>`。
- 响应 `Content-Type: text/event-stream`，**命名事件**：

| 事件名 | data 示例 | 说明 |
|--------|-----------|------|
| start | `{"status":"processing","message":"开始分析..."}` | 流开始 |
| chunk | `{"delta":"这两张命盘..."}` | 增量文本，前端累加 |
| json_block | `{"type":"dimensions","data":{...}}` 或 `{"type":"full","data":{...}}` | 结构化块，可渲染图表 |
| done | `{"status":"completed"}` | 流结束 |
| error | `{"message":"任务不存在或已过期"}` | 错误（任务无效或 AI 异常） |

- 任务一次性消费，过期时间 10 分钟；同一 taskId 不能重复拉流。

**前端示例（EventSource）**:
```ts
const { data } = await request.post('/api/astrology/compatibility/analyze/task', { chartA, chartB });
const taskId = data.taskId;
const es = new EventSource(`/api/astrology/compatibility/analyze/stream?taskId=${taskId}&token=${accessToken}`);
let text = '';
es.addEventListener('chunk', (e) => { const { delta } = JSON.parse(e.data); text += delta; });
es.addEventListener('json_block', (e) => { const { type, data } = JSON.parse(e.data); /* 更新 dimensions 等 */ });
es.addEventListener('done', () => { es.close(); });
es.addEventListener('error', (e) => { /* 解析 e.data 或 e.message */ });
```

**校验点**: 先 POST task 拿到 taskId；GET stream 时带 token；能收到 start → 多次 chunk →（可选）json_block → done；错误时收到 error 事件。

## 4. 深度合盘分析（传统 POST 流）

**请求**: `POST /api/astrology/compatibility/analyze`  
**Body**: `{ chartA, chartB, relationshipType? }`（需 `Authorization: Bearer <token>`）

**响应**: SSE，无命名事件  
- 每条: `data: {"content":"<单块>"}\n\n`  
- 结束: `data: [DONE]\n\n`  
- 错误: `data: {"error":"..."}\n\n`

前端需自行拼接 `content` 得到完整 JSON（同下结构），可解析 summary、compatibility_score、dimensions、guardrails。

完整分析 JSON 结构示例：
```json
{
  "summary": "一句话点评",
  "compatibility_score": 75,
  "dimensions": {
    "communication": { "score": 60, "analysis": "", "advice": "" },
    "values": { "score": 80, "analysis": "", "advice": "" },
    "stress_response": { "score": 50, "analysis": "", "advice": "" }
  },
  "guardrails": ["地雷区：...", "润滑剂：..."]
}
```

## 5. 获取合盘列表

**请求**: `GET /api/astrology/compatibility`

**Query**（均可选）: `limit`（默认 20，最大 100）、`offset`（默认 0）、`keyword`（匹配 name/notes/tags）

**成功响应** (200):
```json
{
  "success": true,
  "data": {
    "archives": [
      { "id": "uuid", "name": "合盘名称", "tags": ["朋友"], "created_at": "2025-02-12T00:00:00.000Z" }
    ],
    "total": 10
  }
}
```

列表不返回 chart_a/chart_b，仅元数据。分页用 `data.archives` + `data.total`。

---

## 6. 获取合盘详情

**请求**: `GET /api/astrology/compatibility/:id`

**成功响应** (200):
```json
{
  "success": true,
  "data": {
    "id": "uuid",
    "chart_a": { },
    "chart_b": { },
    "name": "合盘名称",
    "notes": "备注",
    "tags": ["朋友"],
    "created_at": "2025-02-12T00:00:00.000Z",
    "updated_at": "2025-02-12T00:00:00.000Z",
    "rectification_method": "triangulation",
    "inferred_hour": "chou",
    "confidence": 85,
    "inference_data": { "tags": [], "matchedStars": [] }
  }
}
```

定盘相关字段若无则为 `null`。校验点：能根据 chart_a/chart_b 渲染双盘；若有定盘，可展示 rectification_* 与 inference_data。

---

## 7. 删除合盘存档

**请求**: `DELETE /api/astrology/compatibility/:id`

**成功响应** (200): `{ "success": true, "data": {} }`

无权限或不存在返回 404。

---

## 8. 字段名约定

| 前端 (camelCase) | 请求 Body 可接受 | 响应 data |
|------------------|------------------|-----------|
| chartA | chartA / chart_a | chart_a |
| chartB | chartB / chart_b | chart_b |
| relationshipType | relationshipType / relationship_type | - |
| inferredHour | inferredHour / inferred_hour | inferred_hour |
| inferenceData | inferenceData / inference_data | inference_data |
| - | - | created_at, updated_at, rectification_method, confidence |

---

## 9. 错误响应（统一处理）

- 401: `{ success: false, error: "未认证", message?: string }`
- 400: `{ success: false, error: "参数错误", message?: string }`
- 404: `{ success: false, error: "资源不存在", message?: string }`
- 500: `{ success: false, error: "...", message?: string }`

流式接口在已发送 SSE 后出错时，通过 `data: {"error":"..."}` 传递，前端可解析并提示。

## 10. 前端自测检查清单

**基础 CRUD**
- [ ] 创建：chartA/chartB/name 必填，能拿到 data.id 并跳转/刷新列表
- [ ] 创建：可选传 rectification（method/inferredHour/confidence/inferenceData），详情页能看到对应字段
- [ ] 列表：默认与 limit/offset 分页正常，total 与条数一致
- [ ] 列表：keyword 搜索 name/notes/tags 有效
- [ ] 详情：返回 chart_a、chart_b、定盘四字段，渲染正确
- [ ] 删除：成功后列表更新；无权限/不存在为 404

**深度合盘分析（POST task + GET stream）**
- [ ] POST `/analyze/task` 传 chartA、chartB 得到 taskId
- [ ] GET `/analyze/stream?taskId=xxx&token=xxx` 收到 start → chunk(s) → json_block（可选）→ done
- [ ] chunk 事件累加 delta 得到全文；json_block 的 type=dimensions/full 可渲染图表
- [ ] 任务不存在或过期时收到 error 事件

**深度合盘分析（传统 POST）**
- [ ] POST `/compatibility/analyze` 传 chartA、chartB，按 data.content 拼接得到完整 JSON
- [ ] 能解析并展示 summary、compatibility_score、dimensions、guardrails

**通用**
- [ ] 所有请求带 `Authorization: Bearer <token>`，未登录返回 401
