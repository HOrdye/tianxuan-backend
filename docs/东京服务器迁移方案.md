## 二、迁移步骤

### 阶段 A：旧服务器导出

#### A1. 数据库备份

```bash
# 在旧服务器执行
cd /opt/tianxuan
mkdir -p backups

# 自动获取 PostgreSQL 容器名（旧服务器可能是 my-postgres、tianxuan-postgres-1 等）
PG_CONTAINER=$(docker ps --filter "name=postgres" --format "{{.Names}}" | head -1)
if [ -z "$PG_CONTAINER" ]; then
  echo "错误：未找到 PostgreSQL 容器，请运行 docker ps | grep postgres 确认"
  exit 1
fi
echo "使用容器: $PG_CONTAINER"

BACKUP_FILE="/opt/tianxuan/backups/migration_$(date +%Y%m%d_%H%M%S).sql"
docker exec $PG_CONTAINER pg_dump -U postgres -Fc postgres > "$BACKUP_FILE"

# 验证备份文件（非空且大于 0）
ls -lh "$BACKUP_FILE"
test -s "$BACKUP_FILE" && echo "备份成功" || { echo "备份失败，文件为空"; exit 1; }
```

> `-Fc` 表示自定义格式（压缩），便于跨版本迁移；若需纯 SQL 可用 `-Fp`。

#### A2. 导出后端代码与配置

```bash
# 在旧服务器执行
cd /opt/tianxuan

# 打包后端（排除 node_modules）
tar --exclude='node_modules' --exclude='dist' --exclude='.git' \
  -czvf backups/backend_$(date +%Y%m%d).tar.gz backend/

# 打包 docker 相关（若存在 docker/postgres 目录）
if [ -d "docker" ] && [ -d "postgres" ]; then
  tar -czvf backups/docker_config_$(date +%Y%m%d).tar.gz docker/ postgres/
else
  echo "docker/postgres 目录不存在，跳过打包"
fi

# 复制 .env（含敏感信息，单独传输）
[ -f backend/.env ] && cp backend/.env backups/backend_env_backup.txt
```

#### A3. 传输到新服务器

```bash
# 新服务器需先创建目录：mkdir -p /tmp/migration_files && chmod 777 /tmp/migration_files

# 批量传输（示例 IP: 43.167.239.119）
scp -P 22 /opt/tianxuan/backups/migration_*.sql root@43.167.239.119:/tmp/migration_files/
scp -P 22 /opt/tianxuan/backups/backend_*.tar.gz root@43.167.239.119:/tmp/migration_files/
scp -P 22 /opt/tianxuan/backups/docker_config_*.tar.gz root@43.167.239.119:/tmp/migration_files/ 2>/dev/null || true
scp -P 22 /opt/tianxuan/backups/backend_env_backup.txt root@43.167.239.119:/tmp/migration_files/ 2>/dev/null || true
```

### 阶段 B：新服务器环境准备

#### B1. 安装基础环境

```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 基础工具
sudo apt install -y curl wget git vim htop

# Docker
curl -fsSL https://get.docker.com | sudo sh
sudo systemctl enable docker && sudo systemctl start docker

# Docker Compose
sudo curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose

# Node.js 18+
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt install -y nodejs

# PM2
sudo npm install -g pm2
```

#### B2. 创建目录结构

```bash
sudo mkdir -p /opt/tianxuan/{backend,frontend,docker,postgres,logs,backups,scripts}
sudo chown -R $USER:$USER /opt/tianxuan
cd /opt/tianxuan
```

### 阶段 C：数据库恢复

#### C1. 解压并配置 Docker

```bash
cd /opt/tianxuan
tar -xzvf /tmp/migration_files/docker_config_*.tar.gz

# 如 postgres/data 已有数据且要完全替换，先清空
# rm -rf postgres/data/*
# mkdir -p postgres/data postgres/init
```

#### C2. 创建 docker-compose.yml

```bash
cat > /opt/tianxuan/docker/docker-compose.yml << 'EOF'
version: '3.8'

services:
  postgres:
    image: postgres:17-alpine
    container_name: tianxuan-postgres
    restart: unless-stopped
    environment:
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD}
      POSTGRES_DB: postgres
      PGDATA: /var/lib/postgresql/data/pgdata
    volumes:
      - ../postgres/data:/var/lib/postgresql/data
      - ../postgres/init:/docker-entrypoint-init.d
    ports:
      - "127.0.0.1:5432:5432"
    networks:
      - tianxuan-network
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U postgres"]
      interval: 10s
      timeout: 5s
      retries: 5
    command:
      - "postgres"
      - "-c"
      - "shared_buffers=128MB"
      - "-c"
      - "max_connections=50"
      - "-c"
      - "effective_cache_size=256MB"

networks:
  tianxuan-network:
    driver: bridge
EOF
```

> 若 `command` 段报错，可去掉 `-c` 相关行，使用默认配置。

#### C3. 启动空数据库并恢复

```bash
cd /opt/tianxuan/docker

# 从 backend_env_backup.txt 获取 DB 密码，或生成新密码
# echo "POSTGRES_PASSWORD=新强密码" > .env
echo "POSTGRES_PASSWORD=从旧服务器.env中的DB_PASSWORD复制" > .env

docker-compose up -d

# 等待就绪
sleep 10
docker exec tianxuan-postgres pg_isready -U postgres

# 恢复备份（-Fc 格式用 pg_restore）
docker exec -i tianxuan-postgres pg_restore -U postgres -d postgres --no-owner --no-acl < /tmp/migration_files/migration_*.sql
```

若备份为纯 SQL（`-Fp`）：
```bash
docker exec -i tianxuan-postgres psql -U postgres -d postgres < /tmp/migration_files/migration_*.sql
```

#### C4. 验证数据库

```bash
docker exec tianxuan-postgres psql -U postgres -d postgres -c "SELECT COUNT(*) FROM information_schema.tables WHERE table_schema='public';"
docker exec tianxuan-postgres psql -U postgres -d postgres -c "SELECT COUNT(*) FROM profiles;"
```

### 阶段 D：后端部署

#### D1. 解压并安装依赖

```bash
cd /opt/tianxuan
tar -xzvf /tmp/migration_files/backend_*.tar.gz

# 复制 .env（从备份恢复）
cp /tmp/migration_files/backend_env_backup.txt backend/.env
```

#### D2. 修改 .env（新服务器适配）

```bash
# 编辑 backend/.env
# 必须检查并修改（保持 postgres/postgres 与旧服务器一致）：
# - DB_HOST=localhost
# - DB_PORT=5432
# - DB_NAME=postgres
# - DB_USER=postgres
# - DB_PASSWORD=与 docker/.env 中 POSTGRES_PASSWORD 一致
# - FRONTEND_BASE_URL=新域名或 IP（如有）
# - NODE_ENV=production
```

#### D3. 安装依赖并构建

```bash
cd /opt/tianxuan/backend
npm install
npm run build
```

#### D4. 启动后端

```bash
# PM2 启动
cat > ecosystem.config.js << 'EOF'
module.exports = {
  apps: [{
    name: 'tianxuan-backend',
    script: 'dist/server.js',
    cwd: '/opt/tianxuan/backend',
    instances: 1,
    exec_mode: 'fork',
    env: { NODE_ENV: 'production' },
    max_memory_restart: '500M',
    autorestart: true,
  }]
}
EOF

pm2 start ecosystem.config.js
pm2 startup
pm2 save
```

#### D5. 健康检查

```bash
curl http://localhost:3000/health
# 预期: {"status":"ok","database":"connected",...}
```

### 阶段 E：防火墙与域名

#### E1. 开放端口

```bash
# 若使用 ufw
sudo ufw allow 22
sudo ufw allow 80
sudo ufw allow 443
sudo ufw allow 3000
sudo ufw enable

# 云控制台安全组同样需放行 80/443/3000
```

#### E2. Nginx 反向代理（可选）

若需通过域名访问：

```bash
sudo apt install -y nginx
sudo tee /etc/nginx/sites-available/tianxuan << 'EOF'
server {
    listen 80;
    server_name 你的域名或IP;
    location /api {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
EOF
sudo ln -sf /etc/nginx/sites-available/tianxuan /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

## 三、迁移时间窗口建议

| 阶段 | 预估时间 | 可停机 |
|------|----------|--------|
| A1-A3 导出与传输 | 10-30 分钟 | 否 |
| B 环境准备 | 15-20 分钟 | - |
| C 数据库恢复 | 5-15 分钟 | - |
| D 后端部署 | 5-10 分钟 | - |
| 切换 DNS/流量 | - | 是 |

**推荐**：先在旧服务器完成 A1-A2，将文件传到新服务器。在新服务器完成 B/C/D 并验证后，再切换域名或入口。

---

## 四、回滚方案

若新服务器异常，可快速回退：

1. 将域名 / 入口指回旧服务器
2. 旧服务器保持运行至确认新环境稳定
3. 新服务器问题修复后，可重新执行恢复步骤

## 六、常见问题

**Q: pg_restore 报错 relation already exists**  
A: 备份若为 `-Fc`，可加 `--clean`，会先 drop 再创建。生产环境慎用，建议在空库上恢复。

**Q: 后端连不上数据库**  
A: 检查 `.env` 中 `DB_PASSWORD` 与 `docker/.env` 中 `POSTGRES_PASSWORD` 一致，且 `DB_NAME=postgres`、`DB_USER=postgres`。

**Q: 2G 内存是否够用**  
A: 当前方案（PostgreSQL + Node.js + Nginx）按文档资源分配可在 2G 下运行。若内存吃紧，可适当调小 PostgreSQL `shared_buffers`、`max_connections`。
