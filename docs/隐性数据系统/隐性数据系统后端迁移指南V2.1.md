# 隐性数据系统后端迁移指南 (Backend Migration Guide)

**文档版本**：v2.1 (V1.7 命盘印证笔记版)  
**创建日期**：2026-02-13  
**目标**：构建支持 "命盘印证笔记 (Chart Verification)" 的后端数据底座，实现 "六维雷达" 与 "溯源卡片" 的数据支撑。

---

## 1. 核心变更摘要 (V2.1 Update)

相较于 V2.0，本次更新主要针对前端从 **"3维心理学"** 升级为 **"6维紫微特质"** 的需求。

| 模块 | 旧版 V2.0 | 新版 V2.1 (Current) | 变更原因 |
| :--- | :--- | :--- | :--- |
| **数据模型** | `Psychometrics` (3维) | `Psychometrics` (6维兼容) | 前端雷达图升级为六轴 |
| **Prompt** | 提取 Risk/Price/Load | 提取 6 维紫微特质 | 统一前后端语言体系 |
| **溯源** | 仅存 `source_event` | **增强溯源信息** | 卡片需要显示“来源：事业咨询” |

## 2. API 接口规范

### 2.1 分析并提取隐性信息 (Analyze & Extract)

**Endpoint**: `POST /api/user/implicit-traits/analyze`  
**Auth**: Bearer Token required

**Request Body**:
```typescript
{
  "userQuery": "string",       // 用户提问
  "aiAnalysis": "string",      // AI 回复
  "contextId": "string"        // 会话 ID (必填，用于溯源)
}
```

**Response Body**:
```typescript
{
  "success": true,
  "data": {
    "psychometrics": {
      "decisiveness": 75,  // 决断力 (七杀)
      "empathy": 60,       // 包容心 (天同)
      "drive": 80,         // 行动力 (破军)
      "stability": 40,     // 稳定感 (天府)
      "curiosity": 55,     // 求知欲 (天机)
      "sociability": 65    // 社交力 (贪狼)
    },
    "weighted_interest_tags": [
      { 
        "value": "创业", 
        "weight": 80, 
        "confidence": 0.9, 
        "source_event": "2024-02-13 事业咨询", // 增强的溯源字段
        "last_updated": 1700000000 
      }
    ]
  }
}
```

## 3. 后端核心逻辑实现

### 3.1 Prompt 模板更新 (`ImplicitTraitService.ts`)

请更新 `buildExtractionPrompt` 方法，使其指导 LLM 提取 6 维特质。

```typescript
private static buildExtractionPrompt(
  userQuery: string, 
  aiAnalysis: string, 
  currentTraits: any
): string {
  const currentProfileStr = ImplicitTraitFormatter.formatForExtraction(currentTraits);

  return `你是一个资深的用户心理侧写师。从对话中提取用户画像，并基于紫微斗数特质维度进行量化。

【已知画像】
${currentProfileStr}

【本次对话】
Q: ${userQuery}
A: ${aiAnalysis}

【提取任务】
请分析用户在以下 6 个维度的表现 (0-100分)：
1. **决断力 (Decisiveness)**: 面对风险的果断程度 (七杀/破军特质)
2. **包容心 (Empathy)**: 对他人的共情与接纳 (天同/太阴特质)
3. **行动力 (Drive)**: 执行力与进取心 (破军/廉贞特质)
4. **稳定感 (Stability)**: 内心秩序与情绪稳定性 (天府/天相特质)
5. **求知欲 (Curiosity)**: 逻辑思考与探索意愿 (天机/巨门特质)
6. **社交力 (Sociability)**: 人际交往能力 (贪狼/太阳特质)

【输出 JSON】
{
  "psychometrics": {
    "decisiveness": number,
    "empathy": number,
    "drive": number,
    "stability": number,
    "curiosity": number,
    "sociability": number
  },
  "weighted_interest_tags": [
    { "value": "关键词", "weight": 0-100, "confidence": 0-1, "source_event": "简短的对话摘要，如'关于跳槽的咨询'" }
  ]
}`;
}
```

### 3.2 数据结构定义 (`types.ts`)

更新后端数据库模型定义，支持 6 维字段。建议在 `psychometrics` JSONB 字段中直接存储这些 key。

```typescript
export interface Psychometrics {
  decisiveness: number;
  empathy: number;
  drive: number;
  stability: number;
  curiosity: number;
  sociability: number;
  
  // 兼容旧字段 (可选，如果不再使用可移除)
  risk_tolerance?: number;
  price_sensitivity?: number;
  cognitive_load?: number;
}
```

### 3.3 溯源字段增强

在前端 `TraitCard` 中，我们需要显示 "来源：日常对话分析" 或 "来源：2023-10-15 事业咨询"。
因此，后端在处理 `weighted_interest_tags` 时：

1.  **自动生成 Source**: 如果 LLM 返回的 `source_event` 为空，后端应自动填充格式为 `YYYY-MM-DD ${Topic}` 的字符串。
2.  **Topic 提取**: 建议利用 LLM 顺便提取一个 2-4 字的 "对话主题" (如 "感情困扰", "投资决策") 放入 `source_event`。
