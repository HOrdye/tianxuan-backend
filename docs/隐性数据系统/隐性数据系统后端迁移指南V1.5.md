# 隐性数据系统后端迁移指南 (Backend Migration Guide)

**文档版本**：v2.0 (Detailed Implementation)  
**创建日期**：2026-02-13  
**目标**：将隐性数据提取逻辑从前端 (Thick Client) 迁移至后端 (Thin Client)，实现架构安全与数据一致性。本指南包含可直接复用的核心业务逻辑代码。

---

## 1. 核心架构变更

我们正在从 **Thick Client (前端负责计算)** 转向 **Thin Client (后端负责计算)** 模式。

| 模块 | 旧架构 (Frontend) | 新架构 (Backend) | 职责 |
| :--- | :--- | :--- | :--- |
| **Prompt Engineering** | `ImplicitTraitExtractor.ts` (前端) | **ImplicitTraitService.ts (后端)** | 构建上下文、拼接 Prompt、截断 Token |
| **LLM Integration** | 直接调用 OpenAI/Claude | **Proxy Service** | 隐藏 API Key、计费统计、重试机制 |
| **Business Logic** | Safety Clamp (前端计算) | **Safety Clamp (后端计算)** | 防止画像抖动、异常值过滤 |
| **Data Persistence** | `destinyCardApi.merge` | **Atomic DB Transaction** | 解决并发写入冲突 (Race Condition) |

## 2. API 接口规范

### 2.1 分析并提取隐性信息 (Analyze & Extract)

**Endpoint**: `POST /api/user/implicit-traits/analyze`  
**Auth**: Bearer Token required

**Request Body**:
```typescript
{
  "userQuery": "string",       // 用户在对话中的原始提问 (e.g., "我想问问今年的财运")
  "aiAnalysis": "string",      // AI 占卜师的回复内容 (e.g., "根据卦象...")
  "contextId": "string"        // (Optional) 关联的会话 ID
}
```

**Response Body**:
```typescript
{
  "success": true,
  "data": {
    // 返回更新后的最新画像（后端已完成 Merge）
    "psychometrics": {
        "risk_tolerance": 65,
        "price_sensitivity": 40,
        "cognitive_load": 20
    },
    "weighted_interest_tags": [
        { "value": "wealth", "weight": 70, "confidence": 0.9, "last_updated": 1700000000 }
    ]
  },
  "changes": { // 可选：返回本次变动摘要，用于调试或前端提示
      "risk_delta": +5,
      "new_tags": ["investment"]
  }
}
```

## 3. 后端核心逻辑实现 (TypeScript/Node.js)

以下代码直接移植自前端经过验证的逻辑，请在后端服务中复用。

### 3.1 核心服务类 (`ImplicitTraitService.ts`)

```typescript
import { LLMProvider } from './LLMProvider'; // 假设后端有 LLM 封装
import { UserProfileRepo } from './UserProfileRepo'; // 假设的 DB 访问层
import { ImplicitTraitFormatter } from './ImplicitTraitFormatter'; // 见下文 3.3

export class ImplicitTraitService {

  /**
   * 处理隐性特征分析请求 (Main Entry Point)
   */
  static async analyzeAndMerge(userId: string, userQuery: string, aiAnalysis: string) {
    // 1. 获取当前画像 (Context Retrieval)
    const currentTraits = await UserProfileRepo.getImplicitTraits(userId);

    // 2. 构建 Prompt (Prompt Engineering)
    const prompt = this.buildExtractionPrompt(userQuery, aiAnalysis, currentTraits);
    const systemPrompt = this.getSystemPrompt();

    // 3. 调用 LLM (Model Inference)
    // 建议使用 gpt-4o-mini 或 claude-3-haiku, temperature=0.3
    const responseContent = await LLMProvider.chatCompletion({
        system: systemPrompt,
        user: prompt,
        temperature: 0.3
    });

    // 4. 解析结果 (Parsing)
    const extractedTraits = this.parseExtractionResponse(responseContent);
    if (!extractedTraits) return null;

    // 5. 应用安全钳 (Safety Clamp) - 核心风控逻辑
    if (currentTraits?.psychometrics && extractedTraits.psychometrics) {
        extractedTraits.psychometrics = this.applySafetyClamp(
            currentTraits.psychometrics,
            extractedTraits.psychometrics
        );
    }

    // 6. 原子合并入库 (Atomic Merge)
    // 这一步建议使用数据库事务或 JSONB 原子操作
    const updatedTraits = await UserProfileRepo.mergeTraits(userId, extractedTraits);
    
    return updatedTraits;
  }

  /**
   * 构建提取 Prompt (移植自前端 V1.5)
   */
  private static buildExtractionPrompt(
    userQuery: string, 
    aiAnalysis: string, 
    currentTraits: any
  ): string {
    // 使用 Formatter 格式化当前画像，仅保留 Top 15 标签以节省 Token
    const currentProfileStr = ImplicitTraitFormatter.formatForExtraction(currentTraits);

    return `你是一个资深的用户心理侧写师。从这段用户对话中提取关键画像标签，并基于已知画像进行增量更新。

【已知画像 (Current Profile)】
${currentProfileStr}

【本次对话 (New Dialogue)】
用户问题：${userQuery}
AI分析：${aiAnalysis}

【任务 (Task)】
请基于本次对话修正已知画像。
1. **增量更新**：如果用户的新行为与已知画像冲突（例如已知Risk=Low但询问炒币），请调整权重，而不是直接覆盖。
2. **显性拒绝**：如果用户明确否认某特征（例如"我没有孩子"），将其 confidence 设为 0。
3. **权重定义**：
   - 0-20: 猜测 (Guess)
   - 21-50: 确认 >2次 (Pattern)
   - 51-80: 强意图 (Strong Intent)
   - 81-100: 核心身份 (Verified)

请提取以下信息（输出JSON格式）：
1. weighted_inferred_roles: 角色标签（包含 weight, confidence）
2. weighted_interest_tags: 兴趣标签（包含 weight, confidence）
3. psychometrics: 心理指标
   - risk_tolerance: 0(保守)-100(激进)
   - price_sensitivity: 0(不敏感)-100(敏感)
   - cognitive_load: 0(清晰)-100(焦虑/过载)
4. family_structure: 家庭结构（has_spouse, has_children 等）
5. profession_hints: 职业线索（数组）

输出示例：
{
  "weighted_interest_tags": [
    { "value": "wealth", "weight": 60, "confidence": 0.8, "source_event": "consultation" }
  ],
  "psychometrics": {
    "risk_tolerance": 75,
    "price_sensitivity": 40,
    "cognitive_load": 20
  }
}`;
  }

  private static getSystemPrompt(): string {
    return `你是一个专业的信息提取助手。你的任务是准确、客观地提取信息，不要过度推断。`;
  }

  /**
   * 安全钳 (Safety Clamp) - 防止画像剧烈抖动
   */
  private static applySafetyClamp(current: any, news: any): any {
    const MAX_CHANGE = 20; // 单次会话最大变化幅度
    
    const clamp = (curr: number, target: number) => {
        const delta = target - curr;
        const clampedDelta = Math.max(Math.min(delta, MAX_CHANGE), -MAX_CHANGE);
        return Math.round(curr + clampedDelta);
    };

    return {
        risk_tolerance: clamp(current.risk_tolerance, news.risk_tolerance),
        price_sensitivity: clamp(current.price_sensitivity, news.price_sensitivity),
        cognitive_load: news.cognitive_load !== undefined && current.cognitive_load !== undefined
            ? clamp(current.cognitive_load, news.cognitive_load)
            : news.cognitive_load
    };
  }

  /**
   * JSON 解析与容错
   */
  private static parseExtractionResponse(content: string): any {
    try {
        // 简单的 JSON 清洗逻辑
        const jsonStr = content.replace(/```json\n?|\n?```/g, '').trim();
        const parsed = JSON.parse(jsonStr);
        
        // 必须进行基本的数据结构转换，确保符合 DB Schema
        const now = Date.now();
        const result: any = {};

        // 权重标签标准化
        if (Array.isArray(parsed.weighted_interest_tags)) {
            result.weighted_interest_tags = parsed.weighted_interest_tags.map((t: any) => ({
                value: t.value,
                weight: Number(t.weight) || 0,
                confidence: Number(t.confidence) || 0.5,
                source_event: 'consultation',
                last_updated: now
            }));
        }
        
        if (parsed.psychometrics) {
            result.psychometrics = {
                risk_tolerance: Number(parsed.psychometrics.risk_tolerance) || 50,
                price_sensitivity: Number(parsed.psychometrics.price_sensitivity) || 50,
                cognitive_load: Number(parsed.psychometrics.cognitive_load) || 0
            };
        }

        return result;
    } catch (e) {
        console.error('JSON Parse Error', e);
        return null;
    }
  }
}
```

### 3.2 格式化工具类 (`ImplicitTraitFormatter.ts`)

用于将 DB 中的 JSON 对象格式化为 LLM 可读的字符串。

```typescript
export class ImplicitTraitFormatter {
  static formatForExtraction(traits?: any): string {
    if (!traits) return '无已知画像';

    const lines: string[] = [];

    // 1. 兴趣标签：按权重排序，只取 Top 15 (Token Optimization)
    if (traits.weighted_interest_tags?.length) {
      const topInterests = traits.weighted_interest_tags
        .sort((a: any, b: any) => (b.weight * b.confidence) - (a.weight * a.confidence))
        .slice(0, 15)
        .map((t: any) => `${t.value}(w:${t.weight},c:${t.confidence})`)
        .join(', ');
      lines.push(`- 核心关注: ${topInterests}`);
    }

    // 2. 心理指标
    if (traits.psychometrics) {
      lines.push(`- 心理指标: Risk=${traits.psychometrics.risk_tolerance}, Price=${traits.psychometrics.price_sensitivity}`);
    }

    return lines.length ? lines.join('\n') : '无已知画像';
  }
}
```

## 4. 数据库层实现建议 (Repository Layer)

### 4.1 原子合并策略 (Atomic Merge Strategy)

在 `UserProfileRepo.mergeTraits` 中，请遵循以下合并规则：

1.  **标签列表 (`weighted_interest_tags`)**:
    *   **Upsert 逻辑**: 遍历新标签，如果 `value` 已存在，更新其 `weight`, `confidence`, `last_updated`。如果不存在，append 到数组。
    *   **Limit**: 如果数组长度超过 50，按权重淘汰末位。
    *   **SQL 建议**: 使用 PostgreSQL `jsonb_set` 或应用层先读后写（需加锁 `FOR UPDATE`）。

2.  **心理指标 (`psychometrics`)**:
    *   **Overwrite**: 直接覆盖 `risk_tolerance`, `price_sensitivity` 等字段，因为已经在 Service 层做过 Safety Clamp 计算。

3.  **兼容性字段**:
    *   更新 `weighted_interest_tags` 时，建议同步更新旧的 `interest_tags` (string[]) 字段，以保证旧版 API 兼容性。
