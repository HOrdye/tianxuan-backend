## 📋 问题描述

### 问题1: 解读界面卡牌图片全部显示相同图片

**现象**:
- 所有塔罗卡牌（如"圣杯八"、"圣杯国王"、"权杖七"）都显示相同的图片
- 图片内容为"愚者"（The Fool）的图案
- 卡牌名称和位置信息正确，但图片错误

**影响**:
- 用户无法正确识别抽到的卡牌
- 严重影响用户体验和产品可信度
- 这是**决不可原谅的错误**

### 问题2: 分享图内容排布完全错误

**现象**:
- 分享图布局混乱，不如第一版
- 卡牌图片同样显示错误（所有卡牌相同）
- 内容结构不符合预期

## 🔍 根本原因分析

### 核心问题：前后端卡牌名称格式不一致

**前端标准格式**（`src/features/tarot/utils/storyTarotData.ts`）:
- 大牌：`"The Fool"`, `"The Magician"`, `"The Tower"`（带The）
- 大牌（部分）：`"Strength"`, `"Wheel of Fortune"`, `"Justice"`, `"Death"`, `"Temperance"`, `"Judgement"`（不带The）
- 小牌：`"Seven of Wands"`, `"Eight of Cups"`, `"King of Cups"`（不带The）

**后端返回格式**（`C:\Users\Hordye\Desktop\ssh\tarotPromptGenerator.ts`）:
- Prompt要求：`card_matrix 中的卡牌名称必须使用用户实际抽到的卡牌（英文名，如 "The Tower"）`
- **问题**：后端可能返回的格式与前端不一致，例如：
  - 小牌可能加了"The"前缀：`"The Eight of Cups"` vs `"Eight of Cups"`
  - 大牌可能去掉了"The"：`"Fool"` vs `"The Fool"`
  - 或者返回中文名但格式略有不同

**当前错误处理**（垃圾代码）:
- 前端做了5种模糊匹配策略（精确匹配、规范化匹配、部分匹配、The前缀匹配等）
- 匹配失败时使用占位符（`storyTarotDeck[0]` = 愚者）
- **这是治标不治本的垃圾代码，无法根本性解决问题**

## 🛠️ 明确修复方案

### 方案1：后端统一命名规范（推荐，根本性解决）

**文件**: `C:\Users\Hordye\Desktop\ssh\tarot.service.ts` 或 `tarotPromptGenerator.ts`

**修改内容**:
1. **建立卡牌名称映射表**：基于前端 `storyTarotData.ts` 建立标准名称映射
2. **在返回 `card_matrix` 之前统一规范化**：确保返回的 `card` 字段与前端 `storyTarotData.name` 完全一致
3. **更新Prompt要求**：明确要求AI返回的卡牌名称必须与前端标准格式一致

**代码示例**:
```typescript
// 在 tarot.service.ts 中添加
import { storyTarotDeck } from '@/features/tarot/utils/storyTarotData';

// 卡牌名称规范化函数
function normalizeCardName(cardName: string): string {
  // 查找前端标准名称
  const matchedCard = storyTarotDeck.find(card => 
    card.name === cardName || 
    card.chineseName === cardName ||
    card.name.toLowerCase() === cardName.toLowerCase() ||
    card.name.replace(/^the\s+/i, '') === cardName.replace(/^the\s+/i, '')
  );
  
  if (matchedCard) {
    return matchedCard.name; // 返回前端标准名称
  }
  
  // 如果找不到，记录错误并返回原名称（需要人工处理）
  console.error(`[Tarot Service] 无法规范化卡牌名称: ${cardName}`);
  return cardName;
}

// 在返回结果前规范化
if (result.card_matrix) {
  result.card_matrix = result.card_matrix.map(card => ({
    ...card,
    card: normalizeCardName(card.card) // 统一规范化
  }));
}
```

**或者在后端Prompt中明确要求**:
```typescript
// 在 tarotPromptGenerator.ts 的 generateSpreadPrompt 中
prompt += `**⚠️ 重要：卡牌名称格式要求**\n`;
prompt += `- 大牌（Major Arcana）：必须使用标准英文名\n`;
prompt += `  - 带"The"前缀：The Fool, The Magician, The Tower, The Star, The Moon, The Sun, The World, The Hermit, The Hanged Man, The Hierophant, The High Priestess, The Empress, The Emperor, The Lovers, The Chariot, The Devil\n`;
prompt += `  - 不带"The"前缀：Strength, Wheel of Fortune, Justice, Death, Temperance, Judgement\n`;
prompt += `- 小牌（Minor Arcana）：格式为 "[Number] of [Suit]"\n`;
prompt += `  - 示例：Ace of Wands, Two of Cups, King of Swords, Queen of Pentacles\n`;
prompt += `- 必须与前端标准格式完全一致，不能添加或删除"The"前缀\n`;
```

```typescript
// 简化的匹配函数（只做精确匹配）
const findMatchingCard = (cardName: string): StoryTarotCard | null => {
  if (!cardName) return null;
  
  // 策略1: 精确匹配（中文名）
  let matched = storyTarotDeck.find(card => card.chineseName === cardName);
  if (matched) return matched;
  
  // 策略2: 精确匹配（英文名）
  matched = storyTarotDeck.find(card => card.name === cardName);
  if (matched) return matched;
  
  // 匹配失败：记录错误，返回null
  console.error(`❌ [CRITICAL] 卡牌名称不匹配: "${cardName}"`);
  console.error(`   前端标准格式示例:`, storyTarotDeck.slice(0, 3).map(c => ({ en: c.name, zh: c.chineseName })));
  return null;
};
```
