## 🔍 问题分析

### 问题描述

管理员后台"天机币流水记录"页面显示的数据来自错误的表，导致：
- ❌ 查询的是 `transactions` 表（支付/充值记录）
- ✅ 应该查询 `coin_transactions` 表（天机币流水记录）
- ❌ 返回的数据缺少关键字段：`coin_type`、`old_balance`、`new_balance`、`transaction_type`

### 后端日志证据

```
🔍 [getCoinTransactions Service] 执行数据查询: 
    SELECT 
      t.id,
      t.user_id,
      t.type,
      t.amount,
      t.coins_amount,
      ...
    FROM public.transactions t  -- ❌ 错误：查询的是 transactions 表
    LEFT JOIN public.profiles p ON t.user_id = p.id
    WHERE t.type = ANY($1::text[])
```

### 前端接收到的数据

```json
{
  "id": "eb1f4603-7367-48ea-a4e6-8eed57efc878",
  "user_id": "635510f7-b66d-4f9e-8e47-22b9114a7280",
  "type": "admin_adjust",  // ❌ 这是 transactions 表的字段
  "amount": "0.00",
  "coins_amount": 2500,
  "item_type": "admin_adjustment",  // ❌ 这是 transactions 表的字段
  // ❌ 缺少 coin_type
  // ❌ 缺少 old_balance
  // ❌ 缺少 new_balance
  // ❌ 缺少 transaction_type（coin_transactions 表的字段）
}
```

### 正确的表结构对比

**transactions 表**（支付/充值记录）：
- `id`, `user_id`, `amount`, `coins_amount`, `item_type`, `pack_type`, `status`, `paid_at`, `payment_provider`
- ❌ 没有 `coin_type`、`old_balance`、`new_balance`、`transaction_type`

**coin_transactions 表**（天机币流水记录）：
- `id`, `user_id`, `coins_amount`, `coin_type`, `old_balance`, `new_balance`, `transaction_type`, `description`, `operator_id`, `created_at`
- ✅ 包含所有需要的字段

## 🛠️ 后端修复方案

### 修复位置

后端 `getCoinTransactions` 服务方法（可能在 `src/services/admin.service.ts` 或类似文件）

### 修复前（错误）

```typescript
// ❌ 错误：查询 transactions 表
const query = `
  SELECT 
    t.id,
    t.user_id,
    t.type,
    t.amount,
    t.coins_amount,
    t.item_type,
    ...
  FROM public.transactions t
  LEFT JOIN public.profiles p ON t.user_id = p.id
  WHERE t.type = ANY($1::text[])

`;
```

### 修复后（正确）

```typescript
// ✅ 正确：查询 coin_transactions 表
const query = `
  SELECT 
    ct.id,
    ct.user_id,
    ct.coins_amount,
    ct.coin_type,
    ct.old_balance,
    ct.new_balance,
    ct.transaction_type,
    ct.description,
    ct.operator_id,
    ct.created_at,
    p.email as user_email,
    p.username as user_username,
    op.email as operator_email
  FROM public.coin_transactions ct
  LEFT JOIN public.profiles p ON ct.user_id = p.id
  LEFT JOIN public.profiles op ON ct.operator_id = op.id
  WHERE 1=1
    ${userId ? 'AND ct.user_id = $' + (paramIndex++) : ''}
    ${transactionType ? 'AND ct.transaction_type = $' + (paramIndex++) : ''}
    ${coinType ? 'AND ct.coin_type = $' + (paramIndex++) : ''}
    ${startDate ? 'AND ct.created_at >= $' + (paramIndex++) : ''}
    ${endDate ? 'AND ct.created_at <= $' + (paramIndex++) : ''}
  ORDER BY ct.created_at DESC
  LIMIT $${paramIndex++} OFFSET $${paramIndex++}
`;
```

### 关键修改点

1. **表名**：`transactions` → `coin_transactions`
2. **字段名**：
   - `t.type` → `ct.transaction_type`
   - 添加 `ct.coin_type`
   - 添加 `ct.old_balance`
   - 添加 `ct.new_balance`
   - 移除 `t.amount`、`t.item_type`、`t.pack_type`、`t.status` 等 `transactions` 表特有的字段
3. **WHERE 条件**：
   - `t.type = ANY($1::text[])` → `ct.transaction_type = $X`（如果提供了 transactionType）
   - 添加 `ct.coin_type` 过滤（如果提供了 coinType）
