# 前端页面无法打开（Vite 5174）修复文档

## 现象

- 后端 `npm run dev` 输出“服务器运行在端口 3000”，访问 `http://<服务器IP>:3000` 能返回 JSON。
- 浏览器访问 `http://localhost:5174` / `http://<服务器IP>:5174` 提示找不到页面或无法访问。
- 服务器侧检查监听端口：仅 `3000` 在监听，**`5174` 未监听**。

## 根因分析

- **前端 Vite dev server 并未启动**，或启动失败后未监听 `5174`。
- 或者 Vite 仅绑定在 `127.0.0.1`（默认行为常见），导致外部机器访问 `http://<服务器IP>:5174` 无法连接。
- 另外还可能存在：云服务器安全组/防火墙未放通 `5174`，即便监听 `0.0.0.0:5174` 也无法从公网访问。

## 期望的运行状态（接口契约/网络契约）

- 前端 dev server 需要在服务器上监听：
  - `0.0.0.0:5174`（允许外部访问），或
  - 至少 `127.0.0.1:5174`（仅本机访问；如需外部访问需改为 0.0.0.0 并放通端口）。
- 从服务器本机应满足：
  - `curl -I http://127.0.0.1:5174/` 返回 `200` 或 `304`（通常是 HTML）。
- 从外网访问（若需要公网直连 dev server）应满足：
  - `http://<服务器公网IP>:5174/` 可打开页面。

## 具体修改建议（精确到命令/配置项）

### 1) 启动前端时强制绑定 host

- 在前端项目目录执行：
  - `npm run dev -- --host 0.0.0.0 --port 5174`

说明：Vite 默认可能只监听 `localhost`，外部访问会失败。

### 2) 修改前端 package.json 脚本（推荐）

将 `package.json` 中 `dev` 脚本调整为：

- `vite --host 0.0.0.0 --port 5174`

或（若使用框架脚手架）确保最终传给 Vite 的参数包含：

- `--host 0.0.0.0`
- `--port 5174`

### 3) 修改 Vite 配置（vite.config.ts / vite.config.js）

在 `defineConfig({ server: { ... } })` 中设置：

- `server.host = '0.0.0.0'`
- `server.port = 5174`
- （如需）`server.strictPort = true`

### 4) 云服务器网络放通（若需要公网访问 5174）

- 安全组入站规则：放通 TCP `5174`
- 系统防火墙：放通 `5174/tcp`

注意：如果生产环境不建议直接暴露 dev server 到公网，推荐只在内网使用，或用 Nginx 反代到 80/443。

### 5) 验证命令（服务器侧）

- 查看端口监听：
  - `ss -ltnp | grep 5174`
- 查看进程日志：确认 Vite 启动成功且无报错
- 本机连通性：
  - `curl -I http://127.0.0.1:5174/`

## 与后端联调注意事项（可选）

- 页面打开后，如调用后端 API（3000）报跨域，需要后端 CORS 允许：
  - `Origin: http://<服务器IP>:5174`（或对应域名）

但当前问题属于“前端 dev server 未监听/不可达”，需先解决本文件前述 1-4。
