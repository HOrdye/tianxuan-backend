# 运势页面重构与问题修复计划

本计划旨在解决四大板块显示分离、命盘加载丢失、移除Pro模式以及重构流年/流月内容与UI等5个核心问题。



## 2. 修复命盘数据丢失（一秒后变空状态）（Opt-12）
- **目标**：解决进入命盘界面一秒后显示“还没有生成命盘”的问题。
- **分析**：从截图日志看，报错涉及后端 404 和 `localStorage` 查不到数据。可能是页面挂载后触发了某些校验或同步逻辑（如 `checkUnlockStatus` 或 `fetchUserArchive`），失败后清空了当前选中的 `currentChart`。
- **方案**：
  - 检查 `FortunePlanning.vue`、`useZiweiStore` 或相关路由卫士的生命周期钩子（如 `onMounted`）。
  - 修复同步或校验失败时异常清空 `currentChart` 的逻辑，确保如果本地或内存已有数据，即使后端校验报错也不应导致界面白屏。

## 3. 彻底移除 Pro 模式及专业数据（Opt-13）
- **目标**：删除界面中的“Pro”按钮及所有与“专业数据”相关的内容。
- **方案**：
  - **组件清理**：移除 `FortunePlanning.vue` 顶部的 `⚙ Pro` 按钮。
  - **模板清理**：遍历 `YearlyTheme.vue`、`MonthlyRhythm.vue`、`DecadeOverview.vue`，删除所有 `v-if="proMode"` 的专业数据区块（包括流年干支、流年命宫等毫无内容的表格）。
  - **状态清理**：移除相关的 `proMode` 状态依赖。

## 4. 重构流年内容与 AI Prompt（Opt-14）
- **目标**：根据用户提供的结构（年度主题、核心能量、四化影响、核心课题、关键词、分领域、古人点窍），重构流年 Prompt 及 UI 展示。
- **方案**：
  - **Prompt 优化**：修改 `fortuneNarrativeService.ts`（或 `llmPromptTemplate` 等服务）中生成流年的系统 Prompt，使其严格输出 JSON 格式，并包含上述字段结构。
  - **UI 重构**：修改 `YearlyTheme.vue`（或对应组件），增加相应的卡片区块以展示：核心能量、四化深度影响、核心课题（宜/忌）、分领域提点、古诀点睛等。

## 5. 重构流月内容与 UI 适配（Opt-15）
- **目标**：根据用户提供的新架构（核心主题、四化影响、关键宫位联动、行动指南、古诀点睛）重构流月 Prompt 及展示，同时修复由于页面从侧栏变成单栏导致的宽度失调问题。
- **方案**：
  - **Prompt 优化**：修改流月对应的 Prompt 模板，确保生成符合新结构的 JSON。
  - **UI 宽度与样式优化**：修改 `MonthlyRhythm.vue`，调整卡片的宽度限制（如取消写死的宽度或使其 `width: 100%` 以适应新的单栏大宽屏），并根据新的文本字段重新设计排版（采用卡片化、分模块展示）。
