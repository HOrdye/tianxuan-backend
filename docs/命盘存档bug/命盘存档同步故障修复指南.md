# 命盘存档同步故障修复指南

**创建时间**: 2026年2月12日
**问题描述**: 在 localhost 创建的命盘无法同步到 Network IP 或其他设备。
**根本原因**: 后端服务器 (`49.232.243.107`) 缺少必要的 API 接口 (`/api/astrology/archives`)，导致前端上传失败 (404)。

**本仓库状态**: 已实现 `POST/GET/DELETE /api/astrology/archives` 及 `GET/PUT /api/astrology/archives/:id`，表 `ziwei_chart_archives` 在首次写入时自动创建。若远程仍返回 404，需在 49.232.243.107 部署本仓库最新代码并重启服务。

---

## 🔍 故障确认

您可以使用我们刚刚增强的诊断工具来确认此问题：

1.  打开浏览器控制台 (F12)。
2.  输入并运行：`window.checkBackend()`。
3.  查看 **"步骤4: 测试命盘存档接口"** 的结果。
    *   ❌ 如果显示 **"命盘存档接口不可用"** 或 **"状态码: 404"**，则确认是后端接口缺失。

---

## 🛠️ 解决方案

由于您当前的开发环境是**纯前端**，您无法直接修复此问题，需要根据您对后端的控制权选择以下方案：

### 方案 A：如果您有后端源代码 (Localhost 开发)

如果您在本地有后端项目（Node.js/NestJS/Python等），请执行以下步骤：

1.  **启动本地后端**：
    确保本地后端服务已启动，通常运行在 `http://localhost:3000`。

2.  **修改前端配置**：
    打开 `vite.config.ts`，将代理目标指向本地：
    ```typescript
    // vite.config.ts
    proxy: {
      '/api': {
        target: 'http://localhost:3000', // 修改为本地后端
        changeOrigin: true,
        // ...
      }
    }
    ```

3.  **实现缺失接口**：
    确保您的本地后端已实现了 `POST /api/astrology/archives` 和 `GET /api/astrology/archives`。

### 方案 B：如果您依赖远程服务器 (远程开发)

如果您依赖 `49.232.243.107` 这个远程服务器，您必须联系后端开发人员或部署更新：

1.  **提交需求**：
    将 `docs/后端API实现需求清单.md` 发送给后端开发人员。
    重点指出 **P0 级需求：命盘存档同步 API**。

2.  **接口规范**：
    后端必须实现以下接口才能支持同步：
    *   `POST /api/astrology/archives`：接收前端上传的命盘 JSON 数据。
    *   `GET /api/astrology/archives`：返回用户保存的命盘列表。
    *   `DELETE /api/astrology/archives/:id`：删除存档。

---

## 💡 为什么 Localhost 能看到数据？

这是因为前端使用了 **Local-First (本地优先)** 策略：
1.  当您保存命盘时，数据首先被写入浏览器的 `IndexedDB` 和 `LocalStorage`。
2.  前端随后尝试后台上传到服务器（这一步失败了，报 404）。
3.  在 Localhost 刷新页面时，前端会从本地存储读取数据，所以看起来是“成功”的。
4.  当您切换到 Network IP (手机/其他电脑) 时，那是另一个浏览器环境，本地存储是空的，而服务器又没有数据，所以显示为空。

**结论**：只有修复了后端 API，实现真正的云端同步，才能支持跨设备访问。
